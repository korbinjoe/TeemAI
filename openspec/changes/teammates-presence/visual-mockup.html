Team Activity Strip — Design Rationale

A compact, always-visible team roster that surfaces who is doing what at a glance, without requiring any expansion or navigation.

  • Position: Between mission list and Resources — natural "team awareness" zone. Doesn't push any existing content, uses fixed height (~52px).
  • Avatar circles: Use each agent's signature color from AGENT_COLOR_MAP, matching the existing design system.
  • Status dots: Reuse the project's status color vocabulary (blue=running, yellow=waiting, green=done, gray=idle).
  • Running glow: Active agents get a subtle blue shadow ring, idle agents dim to 50% opacity — visual weight proportional to activity.
  • Header counts: 3 running, 1 waiting, 3 idle — cross-workspace aggregate.
  • Hover tooltip: Shows agent name + current status.
  • Click action: Navigate to /agents page (same as existing Team link).
  • No disruption: Resources and Footer sections are 100% unchanged. Mission list scroll area is unaffected.
  • Cross-workspace: Shows ALL team members across all workspaces, consistent with the sidebar's "immersive overview" positioning.
Integration Impact
  • New component: TeamActivityStrip.tsx (~80 lines)
  • Data source: Aggregate agent status from existing useAllChats + useAgents hooks — no new API needed.
  • Insertion point: MissionSidebar.tsx, between <MissionSessionList/> and <ResourcesSection/>.
  • Height budget: ~52px fixed — does not affect mission list scroll.
  • Existing features: Zero changes to Resources, Footer, or mission list.
